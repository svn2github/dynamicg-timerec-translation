Time Recording - Translation Kit



Setup
  1. Get the latest "strings.xml" file from here: Resources Folder.
    When starting from scratch, use the english (values-en) or german (values-de) strings.xml file as template.
    When fixing/enhancing existing translations, use the according strings.xml file - the folder name pattern is "values-[languageCode]", e.g. values-es for spanish, values-pt for portuguese.
  2. Copy the matching strings.xml file to the top level of your SD Card. Use the "internal SD Card" if you have more than one (e.g. Samsung devices).
  3. Open the app and go to <<Menu / Preferences / Advanced>>. Click on "Translation Kit" and press "OK". Close and reopen the app - you will get an error message if the "strings.xml" file cannot be found.


Translation
  • Contact us at dynamicg.android@gmail.com so that we know which language you're doing the translation for.
  • Edit the "strings.xml" file. Whenever you close/reopen the app, the "strings.xml" file gets reloaded, all changes are reflected immediately.
  • Note you can edit the "strings.xml" directly from your PC: attach the phone via USB, open the File Explorer, go to <<Your Device>> / Internal Storage and edit the "strings.xml" file with Wordpad or any other text editor.
  • Some dialogs are only opened conditionally. See <<Menu / Preferences / Advanced / Translation Kit Extra Dialogs>>
  • When you are done with all texts, send your version of the "strings.xml" file to us so we can integrate it into the app (zip the file so we don't lose the special characters)


Revert
Go to <<Menu / Preferences / Advanced>>. Tap "Translation Kit" to disable it.


Handling of special characters
• <!-- ...text... --> is a comment. ignore these, do not translate
• \n is a newline. "a\nb" will show as "a[newLine]b"
• single and double quotes have to be escaped with backslash: \' or \"
• {1} and {2} are placeholders. "Edit \'{1}\'" may become "Edit 'Tasks'"

• \u.... are utf characters:
\u2026 = ellipsis [...]
\u2192 = right arrow [->]
\u2022 = bullet

• other special characters
&amp; = ampersand [&]
&lt; = less than [<]
&gt; = greater than [>]
